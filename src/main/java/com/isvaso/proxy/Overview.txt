Proxy.
A class that functions as an interface to a particular resource. That resource may be remote,
expensive to construct, or may require logging or some other added functionality.

An interface for accessing a particular resource.

Motivation.
1. You are calling foo.bar();
2. This assumes that foo is in the same process bar();
3. What if, later on, you want to pull all Foo-related operations into a separate process:
   - Can you avoid changing you code?
4. Proxy to the rescue!
   - Same interface, entirely different behavior.
5. This is called a 'communication proxy':
   - Other typesL logging, virtual, guarding, ...
